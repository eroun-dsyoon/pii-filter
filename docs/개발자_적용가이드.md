## 2. 빠른 시작 (30초)

```python
from pii_detector import detect

# 기본 사용
result = detect("전화번호 010-1234-5678, 주민번호 900101-1234567")

print(result.has_pii)       # True
print(result.count)         # 2
print(result.elapsed_ms)    # 0.05
print(result.summary())     # "개인정보 2건 검출 [전화번호, 주민등록번호] (0.050ms)"

# 엔티티 상세
for e in result.entities:
    print(f"  [{e.type_label}] {e.entity} (판별레벨: {e.level_label})")
```

## 3. 함수 시그니처

### `detect(text, level, strict) → PIIResult`

```python
from pii_detector import detect

result = detect(
    text="검사할 텍스트",    # str, 필수
    level=3,               # int (1|2|3), 기본값 3
    strict=True,           # bool, 기본값 True
)
```

#### 파라미터

| 파라미터 | 타입 | 기본값 | 설명 |
|---------|------|--------|------|
| `text` | str | (필수) | 검사할 텍스트. 길이 제한 없음 |
| `level` | int | 3 | 검출 레벨 |
| `strict` | bool | True | 검증 강도 |

**검출 레벨 (`level`)**

| 값 | 이름 | 설명 | 예시 |
|----|------|------|------|
| 1 | 기본 형식 | 표준 포맷만 검출 | `010-1234-5678` |
| 2 | 구분자 변형 | 공백/언더스코어/혼합 구분자 | `010 - 1234 _ 5678` |
| 3 | 대체 문자 | 한글숫자/한자/Leet/이모지/전각 | `공일공-일이삼사-오육칠팔` |

**검증 강도 (`strict`)**

| 값 | 이름 | 설명 | 사용 상황 |
|----|------|------|----------|
| True | 엄격 | 체크섬, 유효 지역코드, BIN 등 검증 | 오탐 최소화 필요 시 |
| False | 형식만 | 자릿수/형식만 맞으면 검출 | 무조건 차단 정책 |

## 4. 출력 구조

### `PIIResult`

```python
@dataclass
class PIIResult:
    has_pii: bool           # PII 존재 여부
    count: int              # 검출 개수
    elapsed_ms: float       # 처리 시간 (ms)
    text_length: int        # 입력 텍스트 길이
    level: int              # 사용된 검출 레벨
    strict: bool            # 사용된 검증 강도
    entities: List[PIIEntity]  # 검출된 엔티티 목록
```

| 메서드 | 반환 | 설명 |
|--------|------|------|
| `.to_dict()` | dict | JSON 직렬화 가능 |
| `.summary()` | str | 사람이 읽기 좋은 한 줄 요약 |

### `PIIEntity`

```python
@dataclass
class PIIEntity:
    type: str               # "PHONE", "RRN", "EMAIL" 등
    type_label: str         # "전화번호", "주민등록번호" 등
    entity: str             # 원본에서 매칭된 문자열
    level: int              # 판별 레벨 (1, 2, 3)
    level_label: str        # "1단계 - 기본 형식" 등
    start: int              # 시작 위치 (0-indexed)
    end: int                # 끝 위치
    normalized: str | None  # 정규화된 값 (대체문자 → 원래 값)
    detail: dict | None     # 추가 정보 (계좌: 은행명 등)
```

### 출력 예시 (JSON)

```json
{
  "has_pii": true,
  "count": 2,
  "elapsed_ms": 0.053,
  "text_length": 35,
  "level": 3,
  "strict": true,

    {
      "type": "RRN",
      "type_label": "주민등록번호",
      "entity": "900101-1234567",
      "level": 1,
      "level_label": "1단계 - 기본 형식",
      "start": 25,
      "end": 39
    }
  ]
}
```

### 계좌번호 `detail` 예시

## 5. 지원하는 PII 유형

| 유형 | type | 형식 | 체크섬 | 엄격 모드 검증 내용 |
|------|------|------|--------|-------------------|
| 주민등록번호 | RRN | YYMMDD-GXXXXXX | O | 생년월일 유효성 + 체크디짓(mod 11) |
| 사업자등록번호 | CRN | XXX-XX-XXXXX | O | 지역코드(100~899) + 체크디짓(mod 10) |
| 전화번호 | PHONE | 0XX-XXXX-XXXX | X | 유효 지역번호 화이트리스트 |
| 여권번호 | PASSPORT | A0000000 | X | 접두사(M,S,R,G,D) + 7자리 |
| 계좌번호 | BANK_ACCOUNT | XXX-XXXX-XXXXXX | X | 자릿수(10~14) + 은행 식별 |
| 신용카드 | CREDIT_CARD | XXXX-XXXX-XXXX-XXXX | O | Luhn 알고리즘 + BIN 범위 |
| 운전면허번호 | DRIVER_LICENSE | RR-YY-SSSSSS-CC | O | 지역코드(01~17) + 체크디짓(mod 97) |
| 이메일 | EMAIL | user@domain.com | X | RFC 5321 형식 |

**전화번호 세부 지원 범위:**

| 유형 | 번호 | 자릿수 |
|------|------|--------|
| 휴대폰 | 010/011/016/017/018/019 | 10~11 |
| 인터넷전화 | 070 | 11 |
| 수신자부담 | 080 | 10~11 |
| 안심번호 | 0502/0504/0505/0507 | 11~12 |
| 대표번호 | 1588/1577/1544 등 | 8 |
| 서울 | 02 | 9~10 |
| 지역 | 031~064 | 10~11 |

## 6. 사용 예시

### 6-1. 기본 검출

```python
from pii_detector import detect

result = detect("연락처: 010-1234-5678")
if result.has_pii:
    for e in result.entities:
        print(f"[{e.type_label}] {e.entity}")
# 출력: [전화번호] 010-1234-5678
```

### 6-2. 레벨별 검출

```python
text = "공일공-일이삼사-오육칠팔"

# Level 1: 미검출 (기본 형식 아님)
r1 = detect(text, level=1)
print(r1.has_pii)  # False

# Level 3: 검출 (대체 문자 인식)
r3 = detect(text, level=3)
print(r3.entities[0].normalized)  # "010-1234-5678"
```

### 6-3. 엄격 vs 형식만

```python
text = "주민번호 990230-1234567"  # 2월 30일 (존재하지 않는 날짜)

# 엄격: 미검출 (유효하지 않은 날짜)
r_strict = detect(text, strict=True)
print(r_strict.has_pii)  # False

# 형식만: 검출 (13자리 형식만 맞으면 OK)
r_loose = detect(text, strict=False)
print(r_loose.has_pii)  # True
```

### 6-4. 배치 처리

```python
from pii_detector import detect_batch

texts = [
    "전화번호 010-1234-5678",
    "오늘 날씨가 좋습니다.",
    "카드번호 4532-0151-2345-6789",
]

results = detect_batch(texts, level=3)
for i, r in enumerate(results):
    print(f"[{i}] {r.summary()}")
# [0] 개인정보 1건 검출 [전화번호] (0.030ms)
# [1] 개인정보 미검출 (0.020ms)
# [2] 개인정보 1건 검출 [신용카드] (0.025ms)
```

### 6-7. 텍스트 마스킹 응용

```python
from pii_detector import detect

def mask_pii(text: str, level: int = 3) -> str:
    """검출된 PII를 마스킹 처리"""
    result = detect(text, level=level)
    # 뒤에서부터 치환 (위치 변경 방지)
    for e in sorted(result.entities, key=lambda x: -x.start):
        masked = f"[{e.type_label}]"
        text = text[:e.start] + masked + text[e.end:]
    return text

original = "홍길동의 전화번호는 010-1234-5678이고 이메일은 hong@test.com입니다."
print(mask_pii(original))
# "홍길동의 전화번호는 [전화번호]이고 이메일은 [이메일]입니다."
```

## 11. 알려진 제한사항

| 항목 | 설명 |
|------|------|
| 이메일 Level 2 | 공백 삽입 이메일(`d s yoon @ eroun . a i`)은 미지원. 영문 문장 오탐 불가피 |
| 여권 Level 3 | Leet 문자(G→6, S→5)가 영문자를 숫자로 변환하여 일부 미탐 |
| 문맥 판단 | 문맥(context) 기반 판단은 미지원. 형식+체계 기반만 |
| 10 이상 문자 | "十一"(11) 같은 2자리 이상 한자/한글 숫자는 미지원 |
